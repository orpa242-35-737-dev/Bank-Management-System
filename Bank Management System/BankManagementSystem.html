OOP Concepts in Java
How this system implements the four pillars of Object-Oriented Programming
Encapsulation
Data Hiding
All account fields (balance, accountNumber, holderName) are declared private. External code must use getBalance() and setBalance(). This prevents invalid direct assignments.
private double balance;
private String accountNumber;

// Only via getter/setter
public double getBalance() {
  return balance;
}
Abstraction
Abstract Class
Account is declared abstract. It defines withdraw() as abstract — forcing every subclass to implement its own withdrawal rules, while hiding the complexity from callers.
abstract class Account {
  // Abstract method — no body!
  public abstract void withdraw(
    double amount
  );
}
Inheritance
Extends Account
SavingsAccount, CurrentAccount, and LoanAccount all extend Account. They inherit deposit(), getBalance(), and other shared methods automatically via super().
class SavingsAccount
  extends Account {

  SavingsAccount(...) {
    super(accNo, name, ...);
  }
}
Polymorphism
Runtime Override
withdraw() is overridden in each subclass. Savings checks minimum balance, Current allows overdraft, Loan treats it as repayment. The correct version is selected at runtime via @Override.
// Same call, different behaviour:
Account a = new SavingsAccount(...);
a.withdraw(500); // checks min bal

Account b = new CurrentAccount(...);
b.withdraw(500); // allows overdraft
